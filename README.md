# Задание 3. Наследование

Напишите класс `Matrix`, реализующий матрицы, и расширяющий его класс `SquareMatrix`, реализующий квадратные матрицы.

## Требования

В классах должны быть определены:

- Конструкторы с параметрами размеров матриц:
  - для `Matrix` — создаёт нулевую матрицу
  - для `SquareMatrix` — создаёт единичную матрицу

- Методы:
  - `Matrix sum(Matrix)` — сумма матриц (должен быть переопределён в `SquareMatrix`)
  - `Matrix product(Matrix)` — произведение матриц

- Методы доступа:
  - `setElement(int row, int column, int value)`
  - `getElement(int row, int column)`

- Метод:
  - `toString()`

## Исключения

Напишите собственный класс исключения, наследующий `RuntimeException`.

Во всех конструкторах и методах должны выбрасываться исключения в случаях, когда операция невозможна (например, при сложении матриц разных размеров).

Исключения должны содержать информацию о проблеме в виде строки, возвращаемой методом `getMessage()`.

## Дополнительные требования

1. Разложить классы по пакетам
2. Использовать `final` там, где это разумно
3. Реализовать метод `equals()` для матриц

## Дополнительное задание

### `MirrorMatrixHor`

Реализуйте класс `MirrorMatrixHor`, расширяющий `Matrix`, для хранения матриц, в которых верхняя половина зеркально отражается в нижней.

- Внутри класса хранится только одна половина матрицы
- Пользователь может обращаться к любому элементу матрицы
- В конструктор передаются реальные размеры матрицы

### Ограничения

- Реализовать с минимальным количеством кода
- Запрещено переопределять методы, кроме:
  - `setElement`
  - `getElement`

### Исключения

Добавить класс исключений `BadMatrixSizesException` (для конструктора)
